# Docker が動く仕組み

## Docker が動く仕組み

Linux はカーネルとユーザランド (userland) から構成される。
ユーザランドはシェル (bash、zesh、...) や基本コマンド (ls、cp、...) などカーネル以外の全てから構成されている。
一定の思想でまとめられたカーネルとユーザランドをディストリビューションといって、Red Hat や Ubuntu などがある。

ユーザランドがソフトウェアからの命令をカーネルに伝え、カーネルが物理的なマシンに命令を伝える。
Docker コンテナにはユーザランドのみが含まれている。
Docker はあくまでホストマシン上の一プロセスとして動作するので、既存のカーネルを使うので BIOS 起動から各種サービス起動までの余計な処理を省け、手軽に起動・終了できる。
